# Reto: Tecnología con Propósito

Objetivo: Identifica una problemática real y documenta tu propuesta de solución en un repositorio de GitHub.

---

## El tema

**"Mujeres resolviendo problemas reales con tecnología."**

Primero, identifica una problemática. Ejemplos:

- Ayudar a estudiantes a encontrar becas
- Salud mental para estudiantes de carreras STEM
- Sostenibilidad en el campus
- Plataforma de networking para mujeres
- Accesibilidad en la educación
- Finanzas personales para estudiantes
- Plataforma de mentoría en STEM

La idea es simple: parte de un problema que conozcan de verdad y piensen cómo la tecnología podría ayudar a resolverlo.

---

## Qué tienen que construir (20 minutos)

Cada equipo debe crear:

1. Un repositorio en GitHub
2. Un README que incluya:
   - Planteamiento del problema
   - Solución propuesta
   - Un wireframe (puede ser en Figma, Canva, Excalidraw, PowerPoint, o incluso dibujado a mano y fotografiado, lo que tengan a la mano)
   - Un roadmap con los próximos pasos
3. Si les alcanza el tiempo, pueden sumar una landing page o un prototipo muy básico. Esto es opcional, no se van a evaluar mal si no llegan a hacerlo.

No es necesario programar nada completo ni funcional. Lo que importa acá es la idea, cómo la comunican y cómo la organizan.

---

## La presentación (5 minutos)

Cada equipo tiene 3 minutos para presentar y responder:

1. ¿Cuál es el problema que están resolviendo?
2. ¿Por qué es importante?
3. ¿Cómo funciona su solución?
4. ¿Qué la hace diferente de lo que ya existe?

---

## Cómo se califica (100 puntos)

| Categoría | Puntos |
|---|---|
| Comprensión del problema | 25 |
| Creatividad | 25 |
| Factibilidad | 20 |
| Organización en GitHub | 15 |
| Presentación | 15 |

---

## Sobre el uso de GitHub

Como parte del reto, el repositorio debe ser el punto central del proyecto de cada equipo. El README.md tiene que incluir, como mínimo:

- Problema
- Solución
- Funcionalidades
- Trabajo futuro
- Integrantes del equipo

---

## Cómo distribuir el tiempo

| Tiempo | Qué hacer |
|---|---|
| 0 a 2 min | El equipo elige su reto y crea el repositorio en GitHub |
| 2 a 17 min | Trabajan en el README: problema, solución, wireframe, roadmap |
| 17 a 20 min | Últimos ajustes y orden del repositorio |
| 20 a 25 min | Presentación (3 min) y preguntas rápidas / calificación |
